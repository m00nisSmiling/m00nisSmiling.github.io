 ကျွန်တော်တို့ Blog Webတစ်ခု ဒါမှမဟုတ် Social Web တစ်ခုခု ကို စပြီး ၀င်တဲ့အခါမှာ ကိုယ့်မှာ ထိုwebsiteအတွက် အကောင့်ဖောက်ပြီးသားဖြစ်မယ်ဆိုရင် 
login or signin ၀င်ရပြီးတော့(login နဲ့ signinက အတူတူပါပဲ)၊ Accountမရှိသေးဘူးဆိုရင် Create or signupဆိုပြီး အကောင့်အသစ်တစ်ခုဖောက်ရပါတယ်။
ဒီ Function(လုပ်ဆောင်ချက်) တွေကို ကျွန်တော့်အနေနဲ့ Securtiyရှုထောင့်က အတိုင်း သေသေချာချာလေး ​ရှင်းပြပါ့မယ်။
ကျွန်တော်တို့ accountတစ်ခုခုကို login၀င်တဲ့အခါမှာ ကျွန်တော်တို့ ဘာကြောင့် login၀င်ဖို့ လိုသလဲ?
ဘာလို့လဲဆိုတော့ ကျွန်တော်တို့က အဲ့ဒီweb appရဲ့ 
ရိုးရိုး၀င်သုံးတဲ့အခါမရနိုင်တဲ့ သုံးခွင့်မရှိတဲ့ functionတစ်ခုခုကို သုံးစွဲချင်လို့ ကျွန်တော်တို့ အကောင့်၀င်ကြရပါတယ်။
(ထားပါတော့ ကျွန်တော်တို့ က comment မန့်ချင်တယ် profile photoပြောင်းချင်တယ် postတင်ချင်တယ် အစရှိသဖြင့်) 
ဒီလိုfunctionတွေဆိုတာ အကောင့် တစ်ခုခုရှိမှာသာ သုံးခွင့်ရမှာပါ၊ accountမရှိပဲနဲ့ သုံးလို့ မရနိုင်ပါဘူး။ 
ဒါကြောင့်မလို့ Userတွေက အကောင့်တစ်ခုဖောက်ပြီး အကောင့်ကို login၀င်ကြရပါတယ်။
ဒီလို Login၀င်တဲ့နေရာမှာ ကျွန်တော်တို့ userတွေက web serverကြီးကို ကျွန်တော်က ဘယ်သူပါ ဘယ်၀ါပါ ဆိုပြီး အသိပေးလိုက်သလိုပါပဲ။
ဥပမာအနေနဲ့ မောင်မောင်က သူ့အ​ကောင့်ကို ၀င်မှသာ web serverကြီးက ဒီuserက မောင်မောင်ပါ ဆိုတာကို သိမှာဖြစ်ပါတယ်။
ဒီတော့ မောင်မောင်က သူ့accountကို၀င်ဖို့ ဘာလုပ်ရသလဲ?
သူ့အကောင့်ရဲ့ MgMgဆိုတဲ့usernameဒါမှမဟုတ်ရင် emailတစ်ခုခုရယ် သူ့ရဲ့password(လျှို့၀ှက်နံပါတ်)ရယ်ကို ရိုက်ထည့်ပြီး Login၀င်လိုက်ရပါတယ်။ 
ဒီလိုမှသာ Web Serverကြီးက ဒီအကောင့်က Mg Mgပါလားဆိုတာကို အသိမှတ်ပြုပြီး
မောင်မောင့်ကို သူနဲ့ဆိုင်တဲ့အကောင့် Mg Mgကို အသုံးပြုခွင့်ပေးပြီး Mg Mgနဲ့ဆိုင်တဲ့postတွေ commentတွေ Profile photoတွေ စတာတွေကို သုံစွဲခွင့်ရှိမှာပါ။
ဒီတော့ အထက်ပါ ဖြစ်စဥ်မှာ အပိုင်း ၂ပိုင်းကို အဓိက ထားပြီးပြောပြချင်တယ်-
ပထမအပိုင်းမှာ web serverကို မောင်မောင်က Username Or Emailနဲ့ Passwordကိုရိုက်ထည့်ပြီး
Web serverရဲ့ အသိမှတ်ပြုမှုကို ရယူရပါတယ်။
ဒီဖြစ်စဥ်ကို authenticationလို့ခေါ်ပါတယ်။
Authenticationဆိုတာက တစ်ခုခုကို ကိုယ်ဘယ်သူဘယ်၀ါ ဖြစ်ကြောင်းပြခြင်းကို ဆိုလိုပါတယ်။
ဒီဖြစ်စဥ်မှာဆိုရင် မောင်မောင်က သူကMgMg accountရဲ့ပိုင်ရှင်ဖြစ်ကြောင်းပြဖို့ အတွက် Usernameရယ် Passwordရယ်ကိုရိုက်ထည့်ရပါတယ်။
Authenticationဆိုတာကို ကားလိုင်စင်လို့ သတ်မှတ်ကြည့်ပါ။ ကားလိုင်စင်ရှိမှ ကားမောင်းလို့ရပါမယ်။ 
ဒီတော့ ကားလိုင်စင်ဆိုတာကြီးက ကိုယ့်ကိုအသိမှတ်ပြုခံရဖို့အတွက် တည်ရှိနေတာ မဟုတ်လား? ဒီတော့ ကားမောင်းလို့ရတာကိုတော့ Authorizationလို့ခေါ်မယ်။
(Authenticationကို ကားလိုင်စင်လို့သတ်မှတ်ပြီး
ကားမောင်းလို့ရတယ်ဆိုတဲ့ အတည်ပြုမှုကြောင့်ရရှိလာတဲ့ အခွင့်အရေးကိုတော့ Authorizationလို့ သတ်မှတ်ပါတယ်။)
ထိုနည်းတူစွာပဲ-
မောင်မောင်က သူဘယ်သူဖြစ်ကြောင်းပါဆိုတာကို ပြဖို့ username and passwordရိုက်ထည့်လိုက်ပြီး အတည်ပြုမှုကိုခံယူလိုက်တာကို Authenticationလို့ခေါ်ပြီး
အတည်ပြုမှုကြောင့်ရလာတဲ့ Postတင်လို့ရတာတွေ account profile photoပြောင်းတာတွေ commentမန့်တာတွေ စတဲ့ 
functionတွေကိုသုံးစွဲခွင့်ရှိသွားတာကိုတော့ Authorizationလို့ခေါ်ပါတယ်။
ဒီလို အမြဲတမ်းပုံသေဖြစ်နေသင့်တဲ့ Authentication And Authorizationဖြစ်စဥ်တွေလဲရှိသလို။ 
တစ်ခါတစ်ရံမှာ Authenticationမလုပ်ပဲနဲ့ Authorizationတန်းရနေတာမျိုးတွေ၊Authorizationတော့မရရှိပေမယ့် 
authenticationတွေက မလုံခြုံတာမျိုးတွေ ဒီလိုမျိုးဖြစ်ရပ်တွေလဲရှိနိုင်ပါတယ်။
ဒီလိုဖြစ်ရပ်တွေကိုတော့ Broken Authentication ၊ Broken Authorizationလို့ ခေါင်းစဥ်တပ်ပြီး ယိုပေါက်တွေအနေနဲ့ သတ်မှတ်ကြလေ့ ရှိပါတယ်။
