WEB SECURITY PENTESTER




×× About Web Security Pentest From Penetration Testing  ××

 Pentesterလို့အတိုကောက်ခေါ်ဆိုကြတဲ့ ဒီ careerက red teamတဲ့တူမယ်လို့ ထင်နိုင်ပေမယ့် objectiveတွေ scopeတွေက
အလွန်ကွဲပြားတာကြောင့်မလို့ Offensiveချင်းတူပေမယ့် careerချင်း မတူညီကြသလို processတွေလဲ မတူကြပါဘူး။
ဒီနေ့မှာ အဓိကအနေနဲ့ offensive securityထဲက penetration testingထဲက Web application securityအကြောင်းကို သေချာရှင်းပြပေးမယ်။

° အဓိက Pentestingရဲ့ objective ရည်ရွယ်ချက်ကဘာလဲ?
Pentester(ထိုးဖောက်၀င်ရောက်သူ) တွေဟာ အဓိကအားဖြင့် system တစ်ခုရဲ့ ဖောက်ထွင်း(exploit) လို့ရနိုင်တဲ့ ယိုပေါက်(vulnerability)တွေကို ရှာဖွေပြီးတော့
၀င်ရောက်(gain access) ပြီး report(မှတ်တမ်း)ရေးပြီး organizationကို ပြန်တင်ပြပေးရတာပဲဖြစ်ပါတယ်။

° Timeframe
အချိန်အားဖြင့် Pentesterတွေရဲ့ processတွေပြုလုပ်ချိန်က အများအားဖြင့် 1dayကနေ 1-10 week(ရက်သက်တစ်ပတ်) အနည်းငယ်ပဲကြာနိုင်ပါတယ်။

° System ဆိုတဲ့အပေါ်ဘယ်လိုမြင်လဲ?
အထက်မှာပြောခဲ့သလိုပဲ Pentesterတွေက systemတွေရဲ့ ယိုပေါက်တွေကိုရှာဖွေပြီး gain accessရှာရယူပြီးတဲ့အခါမှာ
 သက်ဆိုင်ရာ organizationကို ပြန်လည်reportတင်ပြရပါတယ်။
 
ဒီတော့ pentesterတွေ ယိုပေါက်ရှာပေးရတဲ့ systemတွေဆိုတာ ဘာတွေဖြစ်မလဲ?
- Network Services Security
- Applications Security
- Wireless Security
- Physical Security
ဒီလို systemတွေရဲ့security(လုံခြုံရေး) တွေကို အများအားဖြင့်စမ်းသပ်ပြီး ယိုပေါက်ရှာဖွေပေး ကြရတယ်။
- အဲ့systemတွေထဲကနေမှ Applications ဆိုတဲ့ systemတွေကတော့ ရှုပ်ထွေးပြီး Pentestလုပ်ရတာ 
     ခက်ခဲလှသလို အရမ်းလဲ အရေးကြီးပြီး အသုံး၀င်ပါတယ်။

° Applicationမှာ ဘာတွေပါလဲ ဘယ်လိုလေ့လာရမလဲ?
Application Securityမှာဆို အဓိကအားဖြင့်-
- Mobile Application Security နဲ့
- Web Application Security
ဆိုပြီး ၂ပိုင်းခွဲတယ်။
ဒီထဲမှာမှ ပိုမိုကျယ်ပြန့်ပြီး ရှုပ်ထွေးတဲ့ conceptတွေနဲ့ attack တွေအဖြစ်များဆုံး ကတော့
Web application securityမှာပါ။

ဘာလို့လဲ??
ဘာလို့လဲဆိုတော့ ခေတ်အဆက်ဆက် seniorတွေဆီကနေ လက်ဆင့်ကမ်းလာတဲ့စကားအတိုင်းပဲ 
"ကျွန်တော်တို့က web applicationတွေ websiteတွေကို သွားhackမှာ သွားပြီး ယိုပေါက်ရှာဖွေမှာ ဖြစ်တဲ့အတွက်ကြောင့်မလို့ ၊ 
သူတို့တွေ ဘယ်လိုအလုပ်လုပ်သလဲ ဘယ်လိုလည်ပတ်နေသလဲ ဘယ်လိုဖွဲ့စည်းထားသလဲ ဆိုတာတွေကို သိနိုင်သမျှ သိဖို့ လိုအပ်ပါတယ်" ဆိုတော့ ဒါကြောင့်လေ့လာသင့်တာပါ။

2. Kali Linux and Operating System Basic
 ဒုတိယ အဆင့်အနေနဲ့က ငါတို့အနေနဲ့ kali linuxနဲ့ operating systemဆိုတာဘာလဲ ဆိုတာကို သိအောင်လေ့လာသင့်ပါတယ်။
kali linuxကို ဘာကြောင့်လေ့လာဖို့လိုတာလဲဆိုတော့ kali linuxဆိုတာ hacker တွေ pentester တွေ red teamerတွေ စတဲ့ offensive(တိုက်ခိုက်ခြင်း) 
ဘက်ကို ဦးတည်တဲ့ careerအတွက် အသုံးများတဲ့ toolတွေကို default installပေးပြီးသား အလွယ်တကူ installလုပ်လို့ရနိုင်တဲ့ 
open source operating systemတစ်ခုခု ဖြစ်တာကြောင့်မလို့ရယ်၊ နောက်တစ်ခုက ငါတို့ cliလို့ခေါ်တဲ့ command line interfaceကြီးကို ကိုယ်တိုင်ကိုင်တွယ် 
အသုံးပြုတတ်ဖို့ရယ် များစွာသော အကြောင်းပြချက်တွေနဲ့ kali linuxကို သုံးကို သုံးတတ်မှ ဖြစ်ပါလိမ့်မယ်။
ဒါ့အပြင် အချို့သော webတွေမှာ windows serverတွေdebian based linuxတွေ red hat တွေ နဲ့ 
အမျိုးမျိုးသော operating systemပေါင်းစုံနဲ့ ကြုံတွေ့၊ ထိတွေ့လာနိုင်တာကြောင့်မလို့ operating systemဆိုတာဘာလဲ 
ဘယ်လိုအလုပ်လုပ် သလဲ ဆိုတာလောက်တော့ လေ့လာသင့်ပါတယ်။

3. Networking basic or more
ဒီနေရာမှာ အချို့သော Application securityတွေက Network related attackတွေအရမ်းများတာကြောင့်မလို့ 
Networkနဲ့ဆိုင်တဲ့အခြေခံတွေ ရှိရင်လဲကောင်းပါတယ် ဒါမှမဟုတ် levelတစ်ခုထိ ရောက်လေ ပိုကောင်းလေပါပဲ။
ဒါ့အပြင် ipv4 ipv6တွေ subnetခွဲတာတွေ၊ OSI 7 layer လောက်တောင် မရရင်တော့ ဘာမှသုံးစားလို့ရမှာမဟုတ်ပါဘူး။
