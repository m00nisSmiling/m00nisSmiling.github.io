Difference Between Website And Web Application
  Websiteဆိုတာကိုတော့ static(ပုံသေ) ရှိနေတဲ့web serviceတစ်ခုအနေနဲ့သတ်မှတ်တယ်။
Websiteမှာက User Interaction(userထိတွေ့နိုင်တဲ့နေရာ or function) အနည်းငယ်
ဒါမှမဟုတ် အကန့်သတ်နဲ့ ပါ၀င်ပါတယ်။(ဥပမာအနေနဲ့ဆိုCompanyတစ်ခုရဲ့ officialကြေငြှာထားတဲ့ webတွေဆိုရင်
အကြမ်းအားဖြင့် သူတို့မှာ contact us pageတွေပါမယ်၊ သူတို့Companyအကြောင်းကြေငြှာထားတဲ့pageတွေပါမယ်။
ပုံတွေပြပြီး စာတွေရေးထားတဲ့pageတွေပါမယ်။ Userဆီကနေ ratingတောင်းတဲ့pageတွေပါမယ်။  
ဒီလိုမျိုး userအနေနဲ့ တစ်ခုခု actionသွားလုပ်လို့ရတဲ့ functionအနည်းအကျင်းသာ ပါ၀င်တဲ့webတွေကို websiteလို့ခေါ်ဆိုပါတယ်)
Web Applicationဆိုတာကိုတော့ Dynamic(ပုံသေမဟုတ်) ရှိနေတဲ့ web serviceတစ်ခုအနေနဲ့သတ်မှတ်ရပါမယ်။
သူ့မှာဆိုရင် Userကိုယ်တိုင် ထိတွေ့လို့ရတဲ့ ပြောင်းလဲလို့ ၊ဖျက်လို့ ၊အသစ်ဖန်တီးလို့ရတဲ့ Functionတွေပါမယ်။
မတည်ငြိမ်ပြီး ပုံသေမဟုတ်တဲ့ functionတွေပေါ့
(ဥပမာ accountဖွင့်တာလိုမျိုး ၊ account profileပုံပြောင်းတာလိုမျိုး ၊  commentမန့်တာတွေလိုမျိုး အစရှိသဖြင့်ပေါ့)
ဒီလိုမတည်ငြိမ်ပြီး ပြောင်းလဲနေတဲ့functionတွေ ပါ၀င်တဲ့ webကိုတော့ web applicationလို့ခေါ်ဆိုကြရမယ်။(ဥပမာ facebookတို့ twitterတို့လို social webတွေဆိုရင် web applicationလို့ ခေါ်ဆိုကြရပါတယ်)
ဒီတော့ Hackerတစ်ယောက်အနေနဲ့ကြည့်မယ်ဆိုရင် web applicationတွေက websiteတွေထက် ယိုပေါက်များတာတွေ့ရတယ်။
ဘာလို့လဲဆိုတော့ user interactionတွေများများလုပ်လို့ရလေလေ Attackingလုပ်ဖို့ vulnerabilityရှာဖို့ chance များလေလေ ဖြစ်ပါတယ်။(injection payloadတွေထည့်တာလိုမျိုး၊ manual bug huntတာလိုမျိုး)
websiteမှာကျတော့ user interactionအရမ်းနည်းပြီး လုံး၀မရှိသလောက်ဖြစ်တာကြောင့်မလို့ vulnerabilityရှာတွေ့ဖို့ web applicationထက်စာရင် အလွန်နည်းပါးပါတယ်။
ဒါပေမယ့်လဲ Web developerရှုထောင့်ကနေ ကြည့်မယ်ဆိုရင် 
Web applicationတွေကသာ user experienceပိုကောင်းစေတာ လုပ်ငန်းပေါင်းစုံကို functionပေါင်းစုံနဲ့သုံးပြီး
ကိုယ်ပိုင် နာမည်တစ်ခုရသည်အထိ userတွေကို ဆွဲဆောင်နိုင်တာကြောင့်မလို့ ခေတ်နှောင်းပိုင်းမှာ web appတွေကို အသုံးများလာပြီဖြစ်ပါတယ်။
